# Re.form CRM — десктоп-приложение: сборка и обновления

Коротко: пушишь тег версии в GitHub → GitHub сам собирает установщик `.exe` и
кладёт его в «Releases» → в приложении загорается кнопка «Обновить».

---

## 0. Что уже сделано

- База переведена на **SQLite** (отдельный сервер не нужен; файл в
  `%LOCALAPPDATA%\ReformCRM\cosmo.db`).
- `app_desktop.py` — нативное окно (панель + бот внутри).
- `updater.py` + кнопка «Обновить» в панели.
- `reform.spec` (PyInstaller), `installer.iss` (Inno Setup),
  `.github/workflows/build.yml` (авто-сборка).
- Секреты вынесены в `config_local.py` (он в `.gitignore`, в git не попадёт).

---

## 1. Проверка локально (на твоём ПК)

```
pip install -r requirements.txt
python app_desktop.py
```
Откроется окно приложения. (Браузерный режим `python run.py` тоже работает.)

⚠️ Перед публикацией наружу поставь надёжный `ADMIN_PASSWORD` в `config_local.py`.

---

## 2. Залить проект на GitHub (один раз)

1. Установи Git: https://git-scm.com/download/win
2. Создай репозиторий на GitHub: **temanotema/reform-crm** (публичный).
3. В папке проекта:
   ```
   git init
   git add .
   git commit -m "Re.form CRM 1.0.0"
   git branch -M main
   git remote add origin https://github.com/temanotema/reform-crm.git
   git push -u origin main
   ```
4. **ПРОВЕРЬ, что секреты не ушли:** на странице репозитория НЕ должно быть
   файла `config_local.py`. (Если он есть — значит `.gitignore` не сработал,
   напиши мне.)

---

## 3. Собрать установщик (сделать релиз)

Сборка идёт автоматически на серверах GitHub при пуше тега версии:
```
git tag v1.0.0
git push origin v1.0.0
```
Через ~5–10 минут во вкладке **Releases** появится готовый
`ReformCRM-Setup-1.0.0.exe`. Скачай и установи — это и есть приложение.

(Прогресс сборки виден во вкладке **Actions**.)

---

## 4. Как выпускать обновления (в будущем)

1. Внёс изменения в код.
2. Поднял номер в `version.py` (например `1.0.1`).
3. ```
   git add .
   git commit -m "Что изменил"
   git push
   git tag v1.0.1
   git push origin v1.0.1
   ```
4. GitHub соберёт новый установщик. У всех, у кого приложение установлено,
   внизу меню загорится **«Обновить до v1.0.1»** — клик, и оно само скачает
   и поставит новую версию.

> Версия в `version.py` и номер тега должны совпадать (CI подставит версию из
> тега автоматически, так что просто держи их одинаковыми).

---

## 5. Если первая сборка упадёт

Сборка десктоп-приложений иногда требует мелкой подгонки (PyInstaller +
pywebview). Если во вкладке **Actions** сборка покажет ошибку — пришли мне лог
(текст красного шага), и я поправлю `reform.spec` или workflow. Это нормальный
этап, бизнес-логика при этом уже готова и протестирована.

---

## Важное про безопасность

- `config_local.py` (токен бота, пароли, доступы YClients) **никогда** не
  коммить — он в `.gitignore`. В публичном репозитории секретов нет.
- Перед показом/выкладкой смени `ADMIN_PASSWORD` на надёжный.
- Файл базы `cosmo.db` тоже не уходит в git (он в `.gitignore`).
